Background

There are three methods to establish a classifier

A. Model a classification rule directly
B. Model the probability of class memberships given input data
C. Make a probabilistic model of data within each class

Examples

Methoddiscriminative classificationgenerative classificationprobabilistic classificationExamples
AYesNoNok-NN, decision trees, perceptron, SVM
BYesNoYesperceptron with the cross-entropy cost
CNoYesYesnaive Bayes, model based classifiers

Bayes Rule (Bayesian Inference)

  1. P(C∣χ)=P(χ∣C)P(C)P(χ)P(C|\chi)=\frac{P(\chi|C) P(C)}{P(\chi)}P(C∣χ)=P(χ)P(χ∣C)P(C)​

  2. Posterior=Likelihood×PriorEvidencePosterior=\frac{Likelihood \times Prior}{Evidence}Posterior=EvidenceLikelihood×Prior​

    Likelihood=P(χ∣C)Likelihood = P(\chi|C)Likelihood=P(χ∣C)
    Prior=P(C)Prior = P(C)Prior=P(C)
    Evidence=P(χ)Evidence = P(\chi)Evidence=P(χ)
    χ\chiχ is what the data we have now.

    Basic Idea: We got a data χ\chiχ, then we want to know the probability of CCC happened before based what we got (χ\chiχ).

  • Bayes classification
    P(C∣X)∝P(X∣C)P(C)=P(X1,⋅⋅⋅,Xn∣C)P(C)P(C|\Chi) \propto P(\Chi|C)P(C)=P(X_1,\cdot\cdot\cdot, X_n | C) P(C)P(C∣X)∝P(X∣C)P(C)=P(X1​,⋅⋅⋅,Xn​∣C)P(C)
    Difficulty: learning the joint probability P(X1,⋅⋅⋅,Xn∣C)P(X_1,\cdot\cdot\cdot ,X_n|C)P(X1​,⋅⋅⋅,Xn​∣C)

    Assumption that all input features are conditionally independent!

Relevant Issues

  • Violation of Independence Assumption
    • For many real world tasks, P(X1,⋅⋅⋅,Xn∣C)≠P(X1∣C)⋅⋅⋅P(Xn∣C)P(X_1,\cdot\cdot\cdot,X_n|C) \neq P(X_1|C)\cdot\cdot\cdot P(X_n|C)P(X1​,⋅⋅⋅,Xn​∣C)=P(X1​∣C)⋅⋅⋅P(Xn​∣C)
    • Nevertheless, naïve Bayes works surprisingly well anyway!
  • Zero conditional probability Problem
    • If no example contains the attribute value
    • In this circumstance, $$ during test
    • For a remedy, conditional probabilities estimated with

      P^(Xj=ajk∣C=ci)=nc+mpn+m\large{\hat{P}(X_j = a_{jk} | C=c_i) = \frac{n_c + mp}{n + m}}P^(Xj​=ajk​∣C=ci​)=n+mnc​+mp​
      nc:n_c:nc​: number of training examples for which Xj=ajkX_j = a_{jk}Xj​=ajk​ and C=ciC=c_iC=ci​
      n:n:n: number of training examples for which C=ciC=c_iC=ci​
      p:p:p: prior estimate (usually, p=1tp = \frac{1}{t}p=t1​ for ttt possible values os XjX_jXj​)
      m:m:m: weight to prior(number of "virtual" examples, m≥1m\ge 1m≥1)

Summary

  • Naïve Bayes: the conditional independence assumption
    • Training is very easy and fast; just requiring considering each attribute in each class separately
    • Test is straightforward; just looking up tables or calculating conditional probabilities with estimated distributions
  • A popular generative model
    • Performance competitive to most of state-of-the-art classifiers even in presence of violating independence assumption
    • Many successful applications, e.g., spam mail filtering
    • A good candidate of a base learner in ensemble learning
    • Apart from classification, naïve Bayes can do more…
